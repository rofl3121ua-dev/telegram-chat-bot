# TeleBotHost и этот проект

Сайт [telebothost.com](https://telebothost.com) рассчитан на **свой язык TBL** (скрипты в консоли, похожий на JavaScript) и встроенные шаблоны. **Готовый Python-бот на `python-telegram-bot` туда «как zip» не загружается** — это другой стек, не твой репозиторий.

## Что можно сделать

1. **Оставить этот бот на хостинге с Python** (Railway, Render, Fly.io, VPS и т.д.) — тогда используй polling или вебхук (см. ниже).
2. **Писать нового бота с нуля на TBL** на TeleBotHost — это по сути **новый проект**, перенос всех команд вручную.
3. Гибрид (редко имеет смысл): оставить Python на VPS, а на TeleBotHost только простые команды — два разных бота с разными токенами.

## Вебхук для обычного Python-хостинга

В проект добавлен режим **`USE_WEBHOOK=1`** (см. `bot.py`, переменные в `.env.example`):

- `USE_WEBHOOK=1`
- `WEBHOOK_URL` — полный `https://.../путь`, как у `setWebhook` (например `https://app.onrender.com/telegram`)
- `PORT` — порт, который отдаёт платформа (часто подставляется сама)
- при необходимости `WEBHOOK_PATH` — если в `WEBHOOK_URL` только домен без пути

Локально и на ПК по-прежнему удобнее **polling** (без `USE_WEBHOOK`).

## База SQLite

На облаке с эфемерным диском укажи `BOT_STATE_DB_PATH` на постоянный том, иначе `bot_state.db` обнулится при перезапуске.
