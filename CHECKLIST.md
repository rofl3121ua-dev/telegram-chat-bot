# Чеклист: бот «готов к бою»

Отмечай по мере выполнения. Порядок можно менять под себя.

## Токен и BotFather

- [ ] Создан бот в [@BotFather](https://t.me/BotFather), токен только в `.env`, не в git
- [ ] **Privacy Mode** выключен (`/setprivacy` → Disable), если бот должен видеть все сообщения в группе
- [ ] Описание бота в BotFather согласовано с тем, что отдаёт API (`set_my_description` при старте) или задано вручную
- [ ] Команды в меню обновлены (после деплоя бот сам выставляет через `set_my_commands`)

## Окружение

- [ ] Скопирован `.env` из `.env.example`, все нужные переменные заполнены
- [ ] `BOT_OWNER_USERNAME` без `@`, совпадает с аккаунтом владельца
- [ ] Каналы для GIF/мемов/secret: бот добавлен и имеет доступ к постам (`copy_message`)
- [ ] При необходимости: `pip install tzdata` на Windows, чтобы работал `BOT_TIMEZONE` (иначе — локальное время сервера, это ок)

## Группа

- [ ] Бот добавлен в группу; при модерации ссылок — права на удаление сообщений (минимум)
- [ ] Закреп / правила чата с кратким «напиши /help» (по желанию)
- [ ] Админы знают про `/autoreply 1|2|3`

## Запуск и устойчивость

- [ ] Виртуальное окружение: `pip install -r requirements.txt`
- [ ] Пробный запуск: `python bot.py`, в логах нет критичных ошибок
- [ ] Автозапуск после перезагрузки ПК/сервера (Windows: планировщик / ваш `install-autostart.ps1`; Linux: systemd и т.п.)
- [ ] Политика перезапуска при падении процесса (тот же планировщик с «при сбое повторить», или systemd `Restart=always`)

## Данные и бэкапы

- [ ] Файл `bot_state.db` (SQLite) включён в резервное копирование, если статистика важна
- [ ] Понятно, где лежит проект и `.env` на машине, где крутится бот

## Безопасность и злоупотребления

- [ ] Репозиторий без `.env` в истории (`.gitignore` проверен)
- [ ] Владелец понимает, что дают команды `/secret`, `/datedu`, `/pokakat`, `/force_poop`
- [ ] При открытой группе: при необходимости ограничить `/kakapair` кулдауном (уже есть) или правилами чата

## После обновления кода

- [ ] Перезапуск бота
- [ ] Быстрая проверка: `/start`, `/help`, одна команда из блока «механика» (`/my_poop` или `/pokushat`)

## Опционально (когда вырастет нагрузка)

- [ ] Логи в файл + ротация
- [ ] Алерт при долгом дауне (внешний пинг или heartbeat)
- [ ] Тесты на критичную логику (`smart_replies`, режимы `/autoreply`)

---

**Минимум «можно показывать людям»:** отмеченные пункты в блоках *Токен*, *Окружение*, *Группа*, *Запуск* и *После обновления*.
